# Content Audit Framework

## Purpose
This framework was created to review existing product education content and identify opportunities to improve clarity, consistency, and usefulness across customer-facing touchpoints.

The audit focused on feature education content across product updates, help resources, onboarding materials, and lifecycle communication.

## Audit Criteria

| Area | Review Question | Notes |
|---|---|---|
| Target User | Who is this content written for? | End user, admin, team lead, new customer, existing customer |
| Content Type | What type of content is it? | Help article, release note, onboarding guide, lifecycle email, in-product message |
| Journey Stage | Where does this content fit in the customer journey? | Onboarding, feature discovery, adoption, retention, expansion |
| User Need | What user problem does this content solve? | Understand, activate, troubleshoot, adopt, expand usage |
| Clarity | Is the language easy to understand? | Avoid internal terminology and unclear product language |
| Structure | Is the content easy to scan? | Headings, steps, examples, screenshots, CTA |
| Duplication | Does similar content already exist elsewhere? | Identify overlap across help center, emails, and product updates |
| Freshness | Is the content current? | Check if feature names, screenshots, and workflows are outdated |
| Next Action | Is the next step clear? | Learn more, try feature, contact support, read related guide |

## Content Status Categories

| Status | Meaning | Recommended Action |
|---|---|---|
| Keep | Content is accurate and useful | Maintain with minor updates |
| Refresh | Content is useful but needs improvement | Rewrite, restructure, or update examples |
| Merge | Content overlaps with another asset | Combine with related content |
| Archive | Content is outdated or no longer useful | Remove from active customer-facing resources |
| Create | A content gap exists | Build a new article, guide, template, or lifecycle touchpoint |

## Example Audit Notes

| Content Asset | Issue Identified | Recommendation |
|---|---|---|
| Feature release note | Focused too much on product capability, not user value | Rewrite with customer problem and use case first |
| Help center article | Steps were accurate but difficult to scan | Add clearer headings and step-by-step formatting |
| Onboarding email | Mentioned feature without explaining when to use it | Add short use case and link to help article |
| In-product tooltip | Used internal terminology | Replace with plain-language user guidance |

## Recommended Workflow

1. Inventory existing product education content.
2. Group content by feature, audience, and journey stage.
3. Review each asset using the audit criteria.
4. Identify gaps, duplicates, and outdated materials.
5. Recommend whether to keep, refresh, merge, archive, or create content.
6. Align proposed updates with product marketing, customer education, and design partners.
7. Document reusable patterns for future product education updates.

## Outcome
This framework helped create a more consistent way to evaluate product education materials and prioritize improvements across customer-facing content.
